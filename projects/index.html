University of Pune

quickCabs - Cab Booking Desktop Application

Individual Project, March 2014

This was a Software Development project that I worked on in the 3rd year of my Computer Engineering. The aim was to build a prototype of a desktop application by following through the steps of a Software Development Life Cycle. I collected the requirements of a web based cab booking application from the perspective of the users. I set about designing the front-end of the application using C# and Visual Studio 2010. For the back-end, to store the vehicle, user, and driver databases, I used Oracle PL/SQL. When a user made a request for a cab from the user interface, the back end would pair the user up with a car to fit their requirements and a driver who was available at that point in time. One of the challenges I faced was in the calculation of the fare. Initially I had created a small matrix of distances between a small set of pickup/drop off locations, but I thought the system of having preset pickup/drop off locations to be very rigid. To overcome this problem, I decided to use the Google Maps Distance Matrix API, which given the starting point and the destination, returned the shortest road distance between them. This helped me improve the flexibility of the system and also saved me the trouble of having to draw up a complex distance matrix myself.

Predicting Survival on the Titanic

Team Project, October - December 2015

In the Fall of 2015, we participated in the Titanic Machine Learning challenge on Kaggle with the aim of predicting survivors of the disaster. We performed exploratory analysis along with pre-processing on the given training set. We implemented 3 predictive models - SVM, Random Forest and Logistic Regression. My role was to implement SVM using Python Scikit learn and Random Forest using R. We achieved a prediction accuracy of 80% with SVM and 82% with Random Forest.

Gun Control Twitter Sentiment Analysis

Individual Project, October - November 2015

Using Python and the Twitter API, I collected 2000 tweets per day over a period of seven days using the #guncontrol in a JSON format. I then performed sentiment analysis on those tweets by classifying them into buckets - Positive, Neutral and Negative. Finally, I created charts using ggplot2 to visually display the results. One of the main challenges that I faced was that a huge number of tweets containing the #guncontrol were news articles and links. These tweets had a sentiment of 50% Neutral. Since there were a lot of such tweets and did not contribute to understanding how individuals felt about the issue, I decided to remove these tweets. Also, I was able to separate the tweets by time-zone, by doing so I learnt that tweets originating from states in the EST and Pacific time-zones were usually positive about gun control (for gun control), whereas those originating from Central and Mountain time-zones were usually negative about gun control (against gun control).

Predicting Article Popularity

Team Project, January - April 2016

Using data from University of California Irvine's Machine Learning Repository published by Mashable, a well-known online news website, we performed exploratory analysis of data to understand the different features in the dataset and their importance to the prediction model. We implemented Logistic Regression and SVM in Apache Spark (using Python) running on top of HDFS to perform the prediction of article shares based on the various features and we observed a significant performance difference between the cloud environment and R for the two algorithms. My role in this project was to set up the Apache Spark and HDFS environment in the virtual machines we were provided, I was also responsible for the implementation of logistic regression using Python on this ecosystem.


Bike Share Rebalancing

Team Project, February - April 2016

Using data from Healthy Ride Pittsburgh, we performed exploratory analysis of data to understand the different features in the dataset and their importance to the prediction model. We implemented Logistic Regression, Decision Trees and Naive Bayes to predict if a given station is balanced at the end of the day. That is, the number of bikes in the station is equal to the number of docks at the station at midnight. We also used ggplot2 to visualize the patterns of ridership and the traffic inflow and outflow between neighborhoods and stations. My role in this project was the design and implementation of the predictive models in R. We achieved a prediction accuracy of 78% with Logistic Regression, 82% with Naive Bayes and 85% with Decision Trees.

VizBBA

Team Project, September - December 2016

Using Data from the Bay Area Bike Share system we created VizBBA. A system to visualize the pattern of bike share usage in the city of San Francisco. We used a flow map with graduated symbols combined with bar charts, circular heat charts, line plots, and a choropleth map. Two of these visualizations are time series visualizations and the remaining two are geo visualizations. The system seeks to answer four questions users may have, mainly -

  • What are the most popular stations and routes?
  • When are most of the rides taken (with regards to week and day)?
  • Does temperature, wind speed and visibility affect the popularity of the bike share system?
  • How are the annual (long-term) subscribers to the Bay Area Bike Share geographically distributed?

The system helps users gain insights on bike share related problems and helps policy makers decide where to build more bike stations by visually analyzing patterns of use. My role in the project was data exploration, curation using R and design and implementation of the barplots and choropleth maps using D3js. This project won awards for-

  • Best Overall Visualization
  • Most Informative
  • Best Interaction Design

and finished as runner-up for -

  • Most Comprehensive
  • Most Creative
  • Best Visual Design
